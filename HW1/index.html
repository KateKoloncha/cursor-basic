Github command

  • git config --global user.name blabla - ім'я користувача
  • git config --global user.email blabal@bla - email користувача
  • git init - ініціює чистий репозиторій
  • git status - показує перелік всіх файлів створених або змінених та їх статус
  • git add - додає всі файли що ми змінили до контроля версій (Перетворює статус файлів з untracked на added)
  • git commit - комітитть всі зміни, які ми зробили
  • git log - показує всю історію комітів, які були створені
  • git checkout - створює нову гілку або переходить до заданої гілки
  • git branch - створюємо і даємо назву основній гілці
  • git push - відправляє всі зміни в наш репозиторій
  • git pull - стягує всі зміни з віддаленого репозиторію в локальний
  • git merge - об'єднує вказану віддалену гілку з основною локальною
  • git clone - дозволяє клонувати створений репозиторій

gitignore файл який вказує елементи, які git повинен ігнорувати

Advanced

  • git revert - відміняє всі зміни з заданного коміту
  • git reset - відміняє всі коміти з репозиторію після заданного
  • git rebase - пебазування змін з одної гілки на іншу
  • git cherry-pick - бере зміни з одного коміту і намагається їх знову відтворити в поточній гілці
  • git stash - зберігає змінені файли в архів незавершених змін
